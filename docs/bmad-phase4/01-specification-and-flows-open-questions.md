# GameAIgents UX Design Step 1 — Open Questions

_Date: 2026-03-20_

These are remaining implementation-level UX questions after the specification-and-flows step. None of them reopen the wedge.

## 1. Compile summary default format
Should the default compile summary lead with:
- task-oriented inspection cards,
- a generated-asset checklist,
- or a hybrid card + file map layout?

## 2. Compare view density
What is the best default compare layout for the solo technical creator:
- semantic-scope cards first,
- split-pane summary + file diff,
- or timeline-first with expandable detail?

## 3. Brief editor assumption labeling
What is the clearest visual distinction between:
- creator-confirmed facts,
- inferred assumptions,
- and out-of-scope risks?

## 4. Recompile confidence communication
Should preservation confidence be shown as:
- plain-language labels,
- a confidence bar,
- or a risk-tier badge system?

## 5. Provenance workspace entry point
Should Provenance & Export be always visible in the main nav, or should it surface contextually after the first compile while remaining globally reachable?

## 6. Playtest guidance action model
Should each observation default to:
- local edit,
- recompile,
- or a creator-selected next-action menu with no default bias?

## 7. Engine inspection shortcuts
What are the most useful first-inspection links for a Godot-first creator after compile:
- scene tree roots,
- input mappings,
- mechanic config,
- or the objective chain shell?
