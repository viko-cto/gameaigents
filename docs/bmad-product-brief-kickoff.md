## Your job

Refine and validate the **authoritative product brief** for GameAIgents.
Do not regress into a generic prompt-to-game framing.

## Critical strategic constraints

1. GameAIgents is a **creator-to-production copilot**, not a black-box autonomous game maker.
2. The north-star workflow is:
   - `Idea Cascade → World Sketch → Engine Compile → Playtest Agents`
3. World models should be used for **bounded ideation / spatial prototyping**, not as the final runtime.
4. Godot and Unity are both strategically important at the thesis level.
5. Differentiation should focus on:
   - coherence
   - editability
   - engine-native export
   - deterministic state
   - playtest intelligence
   - provenance / disclosure / publishing safety
6. Avoid hypey framing like:
   - "make any game instantly"
   - "smaller Genie"
   - "AI replaces devs"

## What must be resolved in this session

1. Who is the first true primary user?
2. What is the first commercial wedge?
3. What is the first non-gimmick deliverable?
4. What belongs in MVP vs later phases?
5. What should the product explicitly NOT try to do in v1?

## Output needed

Produce a refined product brief that is:
- specific enough to guide PRD creation
- narrow enough to avoid scope explosion
- grounded enough to survive contact with implementation

## Additional instruction

Use the installed BMAD + Game Dev Studio context as reference where useful, but remember:
- GameAIgents is a **platform for making games**, not itself just a single Godot game project
- standard product planning remains the primary workflow
- engine-specific thinking is a downstream constraint, not the whole product identity

Desired end state:
A product brief that can immediately feed the PRD phase without reopening the basic strategy debate.
