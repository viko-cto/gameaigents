## Regulatory & IP

### Steam / Platform Policies
- **Steam**: Requires AI disclosure only for pre-made assets in marketing/player-consumable content. Internal AI tool use needs no disclosure
- Disclosures appear on store pages (near bottom); no dedicated filter. Community browser extension "AI Warning for Steam" adds pop-up warnings
- **Steam Next Fest 2026** (Feb 23-Mar 2): Backlash over "AI slop" flooding 3,455 demos — calls for filters growing
- Games with AI disclosures earned $660M (incl. CoD: Black Ops 6)
- No explicit copyright ownership policy changes — developers retain standard ownership per Steam agreements
- **Recommendation for GameAIgents**: Be transparent about AI use; avoid "slop" perception through quality curation

### EU AI Act (Full Enforcement: August 2, 2026)
- **Minimal risk** (most game AI: pathfinding, NPCs, procedural gen) → No obligations
- **Limited risk** (AI chatbots, deepfake cutscenes, synthetic media) → Must disclose/label AI origins
- **High risk** (emotion recognition, behavioral profiling) → Risk management, documentation, conformity assessments required
- **Prohibited**: Subliminal manipulation for microtransactions, social scoring
- **GPAI rules**: If using foundation models, need transparency docs; systemic-risk models (>10^25 FLOPs) need evaluations
- Fines up to €35M or 7% global turnover
- **Proposed Digital Fairness Act** (Q4 2026) could add loot box bans
- **For GameAIgents**: Tool likely falls under minimal/limited risk. Key obligation = transparency labeling for AI-generated assets. Document training data sources

## Cybersecurity

### Prompt Injection & UGC Security
- Prompt injection is #1 threat for AI game tools — malicious inputs can manipulate generative models to bypass safeguards
- **Key defense strategies**:
  - **Zero-Trust IAM**: RBAC, MFA, adaptive auth for UGC upload/modification
  - **AI-driven anomaly detection**: Real-time scanning of asset pipelines, secret detection (750+ types)
  - **Input sanitization**: Validate and sandbox UGC prompts; on-device AI to minimize cloud exposure
  - **Deep File Inspection**: Pre-execution malware blocking for uploaded assets
- Shift-left security: Integrate scans into developer workflows (>40% visibility gains)
- Quantum-safe encryption emerging for long-term asset integrity
- **For GameAIgents**: Must implement robust input validation, content filtering, and sandboxed generation from day one

### Data Protection
- End-to-end encryption for assets in transit/at rest
- Comply with GDPR for user data, AI Act for transparency
- Regular audits, incident response plans for AI breaches
